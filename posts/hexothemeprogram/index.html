hexo主题开发及部署教程

2025-11-02 2026-01-19

本文仅作为个人开发时的碰见的各类情况而总结出来的建议，逻辑混乱敬请见谅，建议根据标题寻找对应情况

推荐：
一步一步地撰写开发：
Hexo主题开发 - ﹏猴子请来的救兵
这是个非常好的教程！

文件树

在你的theme文件夹下右键打开cmd，输入tree /f

你的tree大致应该长这样，但实际可以随意

theme
│_config.yml	#配置
│
├─layout	#布局
│      aside.ejs	#侧边栏
│      index.ejs	#首页
│      layout.ejs	#总布局
│      nav.ejs		#导航栏
│      post.ejs		#正文部分
│
└─source
    ├─css
    │      theme.css	#主题样式
    │      mobile.css	#移动端适配
    │
    ├─fonts		#在这里放置字体文件
    │      fonts1.woff2
    │      fonts2.ttf
    │
    ├─js
    │      theme.js		#主题js
    │      mobile.js	#移动端适配
    │
    └─lib	#外部库
        │  jquery-3.7.1.min.js
        │  jquery.pjax.js
        │
        ├─	#这里放置更多前置库

代码高亮

把自带的高亮关闭：syntax_highlighter: # empty

md语法失效

hexo没有拓展语法，你在诸如typora用的高亮LaTex啥的并没有(请下载插件)(什么，我还没下)

如何正确给AI提问

在多文件下，需要在新会话把文件树发给AI并发送必要的文件，以下是我的提示词：

我正在制作一款hexo的主题，现在我的文件树如下：

这里填充你的tree

我上传了关键的对应文件， 我将对上述的hexo主题咨询你修改方式，现在无需修改方案，我将在后文一步一步咨询问题，如果你知晓了，请回答“明白”

deepseek无法识别ejs文件，建议转为txt，并附上这段话：
鉴于无法直接上传ejs格式文件,我转成了txt,请知晓他们是ejs文件。

另：deepseekv3.2不会优化，修正总是在补充更多而不是删改，结果就是里面有一堆前后覆盖无效的内容。
gemini2.5pro很厉害，会删改，并且简短，但是对于主题的描述实现不敏感（中译英的原因?）导致实现出来的效果与预期略有偏差，可以要求给出参数位置和解释自己改改

AI的代码在vscode上爆红

ejs通常会先被识别为html，而某些写法在html里是错的所以标记为错误

vscode右下角->选择语言模式->ejs

没有就去下扩展

应当注意的规范

  • 全局配置_config.yml好好利用，别把博客名或者背景图啥的写到样式里，请保持elegent
  • pjax部分刷新是保证你的博客不瞎眼的重要工具，但很容易因为啥啥新功能造成奇奇怪怪的失效问题，建议最后再部署（虽然这样调试的时候就会一直刷新整个页面瞎眼）
  • hexo系列问题之部署到github时会删掉README文件.md(我没管，我把更新日志写博客里了)
